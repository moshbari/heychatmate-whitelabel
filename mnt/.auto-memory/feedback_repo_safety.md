Never touch the original HeyChatMate repo. Always clone it to a separate directory and make changes there.

**Why:** Mosh wants to keep the original codebase intact as a reference/fallback.

**How to apply:** Before any code changes, verify you're working on the cloned copy (heychatmate-whitelabel), not the original (heychatmate-repo).
